实例变量

在 ReactJS 中，你可以很容易的在组件内部定义另外的变量：

const Greeting = React.createClass({
  intervalId: null,
  componentDidMount: () => this.intervalId = setInterval(...),
  render: ...
});

实际上，这么做你同样是改变了这个组件的 "state"，只是不触发 re-render， ReasonReact asks you to correctly put these instance variables into your component's state, into Reason refs.

type state = {
  someRandomState: option(string),
  intervalId: ref(option(int))
};

let component = ...; /* remember, `component` needs to be close to `make`, and after `state` type declaration! */

let make = (_children) => {
  ...component,
  initialState: () => {someRandomState: Some("hello"), intervalId: ref(None)},
  didMount: ({state}) => {
    /* mutate the value here */
    state.intervalId := Some(Js.Global.setInterval(...));
    /* no extra state update needed */
    ReasonReact.NoUpdate
  },
  render: ...
};

所有的实例变量 (subscriptions, refs, etc.) 都必须在 state 里面标记成 ref， 不要在 state record 上直接使用 mutable， 使用一个不可变的字段去指向一个 Reason ref。 为什么要这样约束？ 为了更好的管理 side-effects 和 mutations 来准备 concurrent React。 More details here if you're ever interested.
