cloneElement

Signature: let cloneElement: (reactElement, ~props: Js.t({..})=?, 'anyChildrenType) => reactElement

Same as ReactJS' cloneElement. However, adding extra props to a ReasonReact component doesn't make sense; you'd use a render prop. 因此，ReasonReact.cloneElement 只用于处理一些特殊的情况。 For example, data-* attributes aren't syntactically valid as a function label. 下面这样写不能被正确编译：

<div data-payload=1 className="foo" />

你可以使用 cloneElement 来绕过这个情况：

let myElement =
  ReasonReact.cloneElement(
    <div className="foo" />,
    ~props={"data-payload": 1},
    [||]
  );

This will assign the extra data-payload props (untyped, be careful!) onto the div, through a clever, syntactically valid use of Reason's JS object sugar for BuckleScript objects.

对于非 DOM 组件, 你需要使用有效的 prop 命名。

你也可以使用 cloneElement 来模拟 prop spread , 但这在 ReasonReact 中是不鼓励的。
