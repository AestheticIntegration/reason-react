Ciclos de vida

ReasonReact soporta los eventos de ciclo de vida de ReactJS.

didMount: self => unit

willReceiveProps: self => state

shouldUpdate: oldAndNewSelf => bool

willUpdate: oldAndNewSelf => unit

didUpdate: oldAndNewSelf => unit

willUnmount: self => unit

Nota:

  • Hemos eliminado el prefijo component en todos ellos.
  • willReceiveProps requiere devolver un valor de tipo state, no update state (i.e. no NoUpdate/Update/SideEffects/UpdateWithSideEffects). Se sobreentiende que siempre se actualizará el estado en este ciclo de vida. Si ese no es el caso, simplemente devuelve el state anterior que <code>willReceiveProps recibe como argumento.
  • didUpdate, willUnmount y willUpdate no permiten devolver un nuevo state para ser actualizado, para prevenir loops infinitos.
  • willMount no está soportado. Usa didMount en su lugar.
  • didUpdate, willUpdate y shouldUpdate toman oldAndNewSelf record, de tipo {oldSelf: self, newSelf: self}. Estas dos propiedades son equivalentes a componentDidUpdate, prevProps/prevState junto a props/state en ReactJS. Esto es así también en el caso de willUpdate y shouldUpdate.

If you need to update state in a lifecycle event, simply send an action to reducer and handle it correspondingly: self.send(DidMountUpdate).

Algunos cliclos de vida se comportan de forma diferente, como se describe a continuación.

Acceder a props anteriores o siguientes: retainedProps

Un patrón que se utiliza en ocasiones en ReactJS consiste en acceder a las propiedades de un evento de ciclo de vida: prevProps (componentDidUpdate), nextProps (componentWillUpdate), etc. ReasonReact no mantiene una copia de las props anteriores. Para ello, proveemos del API retainedProps:

type retainedProps = {message: string};

let component = ReasonReact.statelessComponentWithRetainedProps("RetainedPropsExample");

let make = (~message, _children) => {
  ...component,
  retainedProps: {message: message},
  didUpdate: ({oldSelf, newSelf}) =>
    if (oldSelf.retainedProps.message !== newSelf.retainedProps.message) {
      /* do whatever sneaky imperative things here */
      Js.log("props `message` changed!")
    },
  render: (_self) => ...
};

Exponemos ReasonReact.statelessComponentWithRetainedProps y ReasonReact.reducerComponentWithRetainedProps. Ambos funcionan como sus homólogos sin retained-props y requieren que especifiques una nueva propiedad, retainedProps (del tipo que prefieras) en la función make de tu component.

willReceiveProps

Tradicionalmente, ReactJS componentWillReceiveProps toma nextProps. No tenemos nextProps en ReasonML, ya que únicamente están disponible las props pasadas como argumentos etiquetados a la función make. Para acceder a las props en este evento, tienes que usar el API retainedProps:

type state = {someToggle: bool};

let component = ReasonReact.reducerComponentWithRetainedProps("MyComponent");

let make = (~name, _children) => {
  ...component,
  initialState: () => {someToggle: false},
  /* just like state, the retainedProps field can return anything! Here it retained the `name` prop's value */
  retainedProps: name,
  willReceiveProps: (self) => {
    if (self.retainedProps === name) {
      ...
      /* previous ReactJS logic would be: if (props.name === nextProps.name) */
    };
    ...
  }
};

willUpdate

Las nextProps en componentWillUpdate son simplemente los argumentos etiquetados en la función make y las current props (aka this.props) son las props que has copiado en retainedProps, accesible via {oldSelf}:

{
  ...component,
  willUpdate: {oldSelf, newSelf} => ...
}

didUpdate

Las prevProps de ReactJS son las mismas que has copiado en retainedProps, disponibles como oldSelf.

shouldUpdate

Equivalente a shouldComponentUpdate en ReactJS.
