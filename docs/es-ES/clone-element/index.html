cloneElement

Signatura: let cloneElement: (reactElement, ~props: Js.t({..})=?, 'anyChildrenType) => reactElement

Same as ReactJS' cloneElement. However, adding extra props to a ReasonReact component doesn't make sense; you'd use a render prop. Por ello, ReasonReact.cloneElement solo se utiliza en casos muy específicos por cuestiones de interoperabilidad. For example, data-* attributes aren't syntactically valid as a function label. El ejemplo siguiente no sería parseable:

<div data-payload=1 className="foo" />

Usarías cloneElement para sortear el problema de la siguiente forma:

let myElement =
  ReasonReact.cloneElement(
    <div className="foo" />,
    ~props={"data-payload": 1},
    [||]
  );

This will assign the extra data-payload props (untyped, be careful!) onto the div, through a clever, syntactically valid use of Reason's JS object sugar for BuckleScript objects.

Para componentes que no sean del DOM, necesitas utilizar nombres de propiedad válidos.

También puedes usar cloneElement para simular la expansión de props, pero no se recomienda en ReasonReact.
