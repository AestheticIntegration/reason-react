Propiedad Personalizada Class/Component

Tus componentes de ReactJS pueden tener propiedades extras:

class HelloMessage extends React.Component {
  static getRandomNumber() {
    return 4;
  };

  getAnswerToLife() {
    return 42;
  };

  render() {
    ...
  }
}

Pero como los componentes en ReasonReact se crean a través de un record (que tiene campos fijos), no puedes añadir campos adicionales a él. Aquí están las soluciones a ese problema.

Propiedades Estáticas de Clase

Simplemente exporta una función o valor independiente:

let component = ...;
let make = ...;

let getRandomNumber = () => 4;

Hazlo sencillo!

Propiedades de Instancia de Clase

Si una propiedad/método de componente no hace referencia a la instancia del componente (véase this en JS), entonces puede ser simplemente una propiedad estática de clase. En ese caso deberías exportarla con un let normal y corriente.

Si aun así, conceptualmente, el componente requiere acceder a this, intenta de todos modos convertilo en un let que recibe un parámetro normal en vez de acceder a this.

Si esta parte no está clara, o no funciona para tu caso en particular, por favor crea una issue!
