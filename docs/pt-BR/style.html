Style

Desde que o CSS-in-JS está na moda, nós recomendamos nossa escolha oficial. Entretanto, para estilos inline, há a API ReactDOMRe.Style.make:

<div style=(
  ReactDOMRe.Style.make(~color="#444444", ~fontSize="68px", ())
)/>

É uma chamada de função rotulada (digitada!) que mapeia para o estilo {color: '#444444', fontSize: '68px'}. Note que make retorna um tipo opaco ReactDOMRe.style que não você não conseguirá modificar. Também expomos uma função ReactDOMRe.Style.combine que recebe dois styles e combina em um.

Escape Hatch: unsafeAddProp

A API Style.make acima verifica seguramente o tipo de cada campo de estilo! No entanto, as vezes não é possivel detectar alguns campos. If that's the case, the type system will tell you that the field you're trying to add doesn't exist. To remediate this, we're exposing a ReactDOMRe.Style.unsafeAddProp to dangerously add a field to a style:

let myStyle = ReactDOMRe.Style.make(~color="#444444", ~fontSize="68px", ());
let newStyle = ReactDOMRe.Style.unsafeAddProp(myStyle, "width", "10px");
