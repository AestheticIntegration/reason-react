Translate

JSX

Reason supporte une syntaxe JSX ! ReasonReact effectue la transformation depuis un appel de fonction agnostique en un appel spécifique ReasonReact via une macro. Pour tirer parti du JSX de ReasonReact, ajoutez {"reason": {"react-jsx": 2} dans votre bsconfig.json (schéma ici).

Uncapitalized

<div foo={bar}> {child1} {child2} </div>

est transformé en

ReactDOMRe.createElement("div", ~props=ReactDOMRe.props(~foo=bar, ()), [|child1, child2|]);

qui est compilé en code JS:

React.createElement('div', {foo: bar}, child1, child2)

Le tag sans propriétés <div /> est transformé en :

ReactDOMRe.createElement("div", [||]);

qui est compilé en

React.createElement('div', undefined)

Notez que ReactDOMRe.createElement est présent uniquement pour un usage interne par le convertisseur JSX. Si vous avez besoin d'une escape-hatch, utilisez plutôt ReasonReact.createDomElement comme expliqué dans la section enfants.

Capitalized

<MyReasonComponent key={a} ref={b} foo={bar} baz={qux}> {child1} {child2} </MyReasonComponent>

est transformé en

ReasonReact.element(
  ~key=a,
  ~ref=b,
  MyReasonComponent.make(~foo=bar, ~baz=qux, [|child1, child2|])
);

Le tag sans propriétés <MyReasonComponent /> est transformé en :

ReasonReact.element(MyReasonComponent.make([||]));

Le make ci-dessus est précisément la fonction make que vous avez vu dans la section précédente.

Notez comment ref et key ont été extraits de l'appel JSX pour être placés dans l'appel ReasonReact.element. ref et key sont réservés en ReasonReact, tout comme en ReactJS. Ne les utilisez pas comme propriétés dans votre composant !

Children

Les enfants en ReasonReact sont totalement typés. Vous pouvez passer n’importe quelle structure de données (du moment que le composant récepteur le permet). Lorsque vous écrivez :

<MyReasonComponent> <div /> <div /> </MyReasonComponent>

Vous passez en fait le tableau [| <div />, <div /> |] aux enfants de MyReasonComponent. Cela signifie aussi que ce qui suit ne fonctionnera pas :

let theChildren = [| <div />, <div /> |];
<MyReasonComponent> theChildren </MyReasonComponent>

Car en fait, cela se traduit par :

let theChildren = [| <div />, <div /> |];
ReasonReact.element(
  MyReasonComponent.make([|theChildren|])
);

Qui encapsule theChildren dans une autre couche du tableau. To solve this issue, Reason has a special children spread syntax:

let theChildren = [| <div />, <div /> |];
<MyReasonComponent> ...theChildren </MyReasonComponent>

Cela passe simplement theChildren sans tableau autour et donne :

let theChildren = [| <div />, <div /> |];
ReasonReact.element(
  MyReasonComponent.make(theChildren)
);

For more creative way of leveraging Reason's type system, data structures and performance to use children to its full potential, see the Children section!
